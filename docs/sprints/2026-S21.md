# Sprint 2026-S21

## release

- `v0.8.0`

## 本轮目标

- 启动 `v0.8.0` 开放接入边界校准，把第三方 Adapter 接入、Adapter + Provider 兼容性判断与真实 provider 验证样本的版本关系写清。
- 明确指定 provider 产品的正式支持不是 Syvert 主线目标。

## 入口事项

- `CHORE-0291-open-adapter-provider-boundary`：校准开放 Adapter 与 provider 兼容性路线，对应 Issue `#291`
- `FR-0023-third-party-adapter-entry-path`：冻结并实现第三方 Adapter-only manifest / fixture / contract test entry 与 SDK docs / migration，对应 FR Issue `#295`
- `FR-0027-multi-profile-resource-requirement-contract`：冻结并实现当前双参考 slice 的 multi-profile resource requirement contract，对应 FR Issue `#294`
- `FR-0024-adapter-capability-requirement-contract`：冻结 Adapter capability requirement 声明契约，对应 FR Issue `#296`
- `FR-0025-provider-capability-offer-contract`：冻结 Provider capability offer 声明契约，对应 FR Issue `#297`
- `FR-0026-adapter-provider-compatibility-decision`：冻结 Adapter requirement x Provider offer 的 compatibility decision、fail-closed 与 no-leakage 边界，对应 FR Issue `#298`

## 进入前依赖

- `v0.7.0` 已正式发布，adapter-owned provider port 与 native provider 拆分已完成。
- `FR-0021` 的历史范围保持不变：不接入外部 provider，不引入 provider selector / fallback。

## 目标判据

- roadmap、Adapter SDK 与框架定位文档一致表达 `v1.0.0` 前的开放接入能力目标。
- `v0.8.0` 只建立兼容性判断与开放接入路径的 planning truth，不声明指定 provider 产品正式支持。
