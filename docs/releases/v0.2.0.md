# Release v0.2.0

## 目标

- 把 `v0.1.0` 的“能跑”推进到“可验证”，为共享输入模型、错误模型、adapter registry、contract harness 与版本 gate 建立稳定 requirement 基线。

## 明确不在范围

- 不在仓库内维护 backlog、project 或 sprint 状态镜像
- 不把 GitHub `Phase / FR / Work Item` 重新复制成仓内第二套调度层
- 不提前进入 `v0.3.0` 的持久化、分布式执行或资源系统能力
- 不把版本 gate 固定为某个唯一脚本、CI 文件或工作流编排

## 目标判据

- `#63` 已作为 `v0.2.0` 当前 Phase 承载 `FR-0004`、`FR-0005`、`FR-0006`、`FR-0007`
- `#63` 下的 `FR-0004`、`FR-0005`、`FR-0006`、`FR-0007` 已在 GitHub 建立为 `v0.2.0` requirement 容器，formal spec 按各自节奏逐项推进
- 版本 gate 已被明确定义为 `v0.2.0+` 的固定版本流程，且覆盖 contract harness、双参考适配器回归与平台泄漏检查
- GitHub 调度层与仓内 release / sprint / formal spec / exec-plan 索引保持一致
