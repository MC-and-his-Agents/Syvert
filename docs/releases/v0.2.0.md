# Release v0.2.0

## 目标

- 把 `v0.1.0` 的“能跑”推进到“可验证”，并在进入该版本主线实现前先收敛 GitHub 调度层与仓内语义层的治理契约。

## 明确不在范围

- 不在仓库内维护 backlog、project 或 sprint 状态镜像
- 不把 GitHub `Phase / FR / Work Item` 重新复制成仓内第二套调度层
- 不在本轮治理收敛中改造与 `TODO.md` 清理无关的 harness 运行时行为

## 目标判据

- GitHub 调度层已统一为 `Phase -> FR -> Work Item`
- 仓库语义层已统一为 formal spec、exec-plan、风险、验证证据、checkpoint 与恢复上下文
- `FR-0003` formal spec 已入库，且明确 formal spec 绑定 FR、Work Item 为唯一执行入口
- `FR-0004` formal spec 已入库，且冻结 `InputTarget` 与 `CollectionPolicy` 的共享模型边界
- legacy `TODO.md` 已退出 formal spec live 最小套件、模板与治理主链路

## 纳入事项

- `FR-0003-github-delivery-structure-and-repo-semantic-split`：`v0.2.0` governance requirement 主线，对应 Issue `#55`
- `FR-0004-input-target-and-collection-policy`：`v0.2.0` 共享输入模型与采集策略模型，对应 Issue `#64`
- `GOV-0027-governance-contract-rewrite`：治理契约重写与口径收敛，对应 Issue `#56`
- `GOV-0028-harness-compat-migration`：后续 harness 兼容迁移，对应 Issue `#57`
- `GOV-0029-remove-legacy-todo-md`：移除 legacy `TODO.md` 的正式治理流入口与存量 compat 残留，对应 Issue `#58`

## 相关前提

- `#54` 作为 `pre-v0.2.0 kickoff governance convergence` 的 Phase 已在 GitHub 建立
- `vision.md` 与 `docs/roadmap-v0-to-v1.md` 仍是阶段边界的上位来源
