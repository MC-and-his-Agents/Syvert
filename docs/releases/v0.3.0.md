# Release v0.3.0

## 目标

- 建立最小任务与结果持久化闭环，让任务在执行完成后仍可沿共享 Core 路径被查询与回读。

## 明确不在范围

- 不在仓库内维护 backlog、project 或 sprint 状态镜像
- 不引入 HTTP API、分布式工作队列或高级任务控制
- 不在本 release 内展开丰富查询层或资源系统

## 目标判据

- `FR-0008` formal spec 已入库，并冻结任务状态、终态结果、执行日志、共享序列化与本地持久化边界
- `FR-0008` 的实现子事项已把共享任务记录模型与本地持久化管线合入主干
- `FR-0009` 已把 CLI 查询任务状态/结果与同路径执行闭环合入主干
- 任务在执行完成后仍可查询，且状态、结果与日志语义保持一致
- 持久化没有绕过 Core 内部机制
