# Release v1.6.0

## 目标

- 发布 `#445` Batch / Dataset Core Contract release slice，完成 batch request / target set、item-level outcome、partial success/failure、resume token、batch audit trace、dataset record / sink 最小模型、readback 与 audit replay。
- 复用 `v1.3.0`、`v1.4.0`、`v1.5.0` read-side result envelopes，不重新定义 creator/comment/media/content 私有字段。
- 明确 `#382` 只是 deferred roadmap anchor，不是 execution entry、completed truth 或 release version source。

## 明确不在范围

- 不交付 scheduler / scheduled execution。
- 不交付 write-side operations。
- 不交付 content library、BI / analytics product、UI 或上层 workflow。
- 不交付 provider selector、fallback、marketplace、provider ranking、provider SLA 或 product support list。
- 不提交 raw payload files，不记录 source names、本地路径、storage handles、private account/media/creator fields。

## 目标判据

- FR `#445` formal spec 已冻结 Batch / Dataset Core public vocabulary 与 fail-closed 边界。
- Runtime carrier 已稳定承载 batch request、item outcome、resume token、dataset record 与 reference sink。
- TaskRecord/result query/runtime admission/compatibility consumers 已证明 batch/dataset public carriers 可消费，且不读取 platform-private result fields。
- Sanitized evidence 已覆盖 partial success、partial failure、all failed、resume、duplicate target、dataset replay 与 resource boundary。
- Closeout Work Item 承接 release/sprint/FR/Phase truth reconciliation 和 explicit release decision。
