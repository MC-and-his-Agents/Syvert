# Release v1.2.0

## 目标

- 发布 Resource Governance Foundation，使 `account` / `proxy` 资源生命周期具备凭据材料边界、会话健康证据、健康准入、Core-owned invalidation 与 consumer no-leakage 基础。
- 保持 `v1.1.0` operation taxonomy baseline 与 `content_detail_by_url` stable runtime path 不变。
- 证明真实 Adapter 场景中账号凭据 / 会话健康不能继续只依赖 opaque material，而必须进入 Core-governed admission evidence。

## 明确不在范围

- 不发布自动登录、自动刷新、自动修复循环或账号矩阵产品能力。
- 不新增资源类型，不把平台私有 cookie/token/header/session 字段暴露给 AdapterRequirement、ProviderOffer、registry discovery、TaskRecord public envelope 或 Core routing metadata。
- 不新增 provider selector、fallback、priority、ranking、marketplace、provider product SLA 或 health SLA。
- 不发布新的 executable runtime operation、上层应用 workflow 或 Python package artifact。

## 目标判据

- FR `#387` formal spec suite 已合入并冻结 `CredentialMaterial`、`SessionHealth`、`ResourceHealthEvidence`、resource lease / invalidation / admission 边界。
- Runtime carrier 已实现 additive health admission，默认 `acquire()` 行为不变；只有显式要求健康 account session 时才 fail-closed 消费 health evidence。
- Active lease invalidation 仅在 invalid evidence 绑定 active lease 时由 Core 执行；pre-admission invalid 无 active lease 只拒绝 admission，不直接改 `AVAILABLE -> INVALID`。
- AdapterRequirement、ProviderOffer、CompatibilityDecision 与 public metadata no-leakage guard 禁止 credential/session/private health 字段泄漏。
- Evidence artifact 可复验证明 healthy/stale/unknown/invalid_contract/pre-admission invalid/active invalidation/public projection redaction 场景成立。
- `account` / `proxy` lifecycle、trace、bootstrap、real adapter regression、CLI/API same path、provider/platform no-leakage 回归通过。
