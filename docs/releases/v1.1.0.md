# Release v1.1.0

## 目标

- 发布 Operation Taxonomy Foundation，为 v1.x runtime capability 扩展建立统一命名、生命周期、准入和兼容性判断基础。
- 保持 `v1.0.0` stable baseline 不变：唯一 stable runtime slice 仍是 `content_detail / content_detail_by_url / url / hybrid`。
- 将候选能力族作为 `proposed` taxonomy entries 表达，但不交付可执行 runtime capability。

## 明确不在范围

- 不发布 `content_search`、`content_list`、`comment_collection`、`creator_profile`、`media_asset_fetch`、`media_upload`、`content_publish`、`batch_execution`、`scheduled_execution` 或 `dataset_sink` runtime capability。
- 不新增 provider selector、fallback、priority、ranking、marketplace、provider product SLA 或平台私有业务对象。
- 不新增上层应用 workflow。
- 不发布 Python package artifact。

## 目标判据

- FR `#368` formal spec suite 已合入并冻结。
- Runtime taxonomy registry、validator、stable lookup 与 proposed candidate fail-closed 规则已合入。
- Adapter requirement、Provider offer、Adapter/Provider compatibility decision 已消费 taxonomy stable lookup，且 v1.0.0 行为不变。
- Admission evidence 证明 proposed `content_search` 与 `comment_collection` 可表达但不能执行、不能 matched。
- `content_detail_by_url` 双参考 baseline、第三方 Adapter contract entry、CLI/API same path、provider no-leakage 与 platform leakage gate 通过。
