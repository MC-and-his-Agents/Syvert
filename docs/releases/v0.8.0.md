# Release v0.8.0

## 目标

- 清理 Core / Adapter / Provider 边界，使 `v1.0.0` 前的开放接入目标可被一致解释。
- 稳定第三方 Adapter 接入路径，明确 Adapter-only 与 Adapter + Provider 两种接入形态的职责边界。
- 冻结 Adapter capability requirement 与 Provider capability offer 的兼容性判断模型，为后续真实 provider 验证样本提供 formal spec 入口。

## 明确不在范围

- 不正式支持任何指定外部 provider 产品。
- 不引入 Core provider registry、provider selector、provider fallback priority 或 provider marketplace。
- 不承诺某一个 provider 覆盖所有 Adapter capability。
- 不改写 `v0.7.0` release truth、`FR-0021` formal spec 或既有 provider port evidence。

## 目标判据

- roadmap、Adapter SDK 与定位文档一致表达：社区接入 Syvert 的主入口仍是 Adapter。
- Adapter + Provider 接入必须通过兼容性判断，而不是通过 Core 感知具体 provider 产品。
- 后续 `v0.9.0` 可在该兼容性判断上执行至少一个真实 provider 验证样本。
