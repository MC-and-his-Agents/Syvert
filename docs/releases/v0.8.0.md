# Release v0.8.0

## 目标

- 清理 Core / Adapter / Provider 边界，使 `v1.0.0` 前的开放接入目标可被一致解释。
- 稳定第三方 Adapter 接入路径，明确 Adapter-only 与 Adapter + Provider 两种接入形态的职责边界。
- 冻结 Adapter capability requirement 与 Provider capability offer 的兼容性判断模型，为后续真实 provider 验证样本提供 formal spec 入口。

## 明确不在范围

- 不正式支持任何指定外部 provider 产品。
- 不引入 Core provider registry、provider selector、provider fallback priority 或 provider marketplace。
- 不承诺某一个 provider 覆盖所有 Adapter capability。
- 不改写 `v0.7.0` release truth、`FR-0021` formal spec 或既有 provider port evidence。

## 目标判据

- roadmap、Adapter SDK 与定位文档一致表达：社区接入 Syvert 的主入口仍是 Adapter。
- Adapter + Provider 接入必须通过兼容性判断，而不是通过 Core 感知具体 provider 产品。
- 后续 `v0.9.0` 可在该兼容性判断上执行至少一个真实 provider 验证样本。

  - 状态：`#299/#300/#301/#302` 已合入主干；`#303` 负责父 FR closeout。
- `FR-0024-adapter-capability-requirement-contract`：冻结 Adapter capability requirement 声明契约，对应 FR Issue `#296`
  - 状态：`#313/#314/#315` 已合入主干；`#316` 负责父 FR closeout，并把 `FR-0024` 作为 `#298 / FR-0026` 的 Adapter requirement input truth。
- `FR-0025-provider-capability-offer-contract`：冻结 Provider capability offer 声明契约，对应 FR Issue `#297`
  - 状态：`#319` 负责 formal spec closeout；`#320/#321/#322` 负责 validator、SDK docs / evidence 与父 FR closeout。
- `FR-0026-adapter-provider-compatibility-decision`：冻结 Adapter requirement x Provider offer 的 compatibility decision、fail-closed 与 no-leakage 边界，对应 FR Issue `#298`
  - 状态：`#323` 负责 formal spec closeout；`#324/#325/#326/#327` 负责 runtime、no-leakage guard、docs / evidence 与父 FR closeout。

## 相关前提

- `v0.7.0` 已完成 adapter-owned provider port 与 native provider 拆分。
- `FR-0021` 仍只批准仓内内部边界，不批准外部 provider 接入。
