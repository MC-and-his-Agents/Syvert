# Release v0.1.0

## 目标

- 证明同一 Core 契约能够承载真实参考适配器，并为治理与执行工件建立最小稳定信息架构。

## 明确不在范围

- 不在仓库内维护 backlog 或 sprint 状态镜像
- 不引入独立于既有治理入口之外的新自动化门禁产品面
- 不扩展超出双参考适配器验证切片之外的业务实现或平台特定能力

## 目标判据

- 治理协议支持 `Issue / item_key / item_type / release / sprint` 事项上下文
- 结构层提供 `release` / `sprint` 聚合索引与 `exec-plan` 模板入口
- 同一 Core 可运行小红书与抖音两个真实参考适配器
