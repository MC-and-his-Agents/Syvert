# Release v1.4.0

## 目标

- 发布 `#404` 首个 comment-collection batch，完成 `comment_collection` 的 formal spec、runtime carrier、consumer migration 与 sanitized evidence。
- 复用 `v1.3.0` 的 read-side collection foundation，同时冻结 comment-specific hierarchy、visibility、reply cursor 与 normalized comment item public surface。
- 保持 `content_detail_by_url` stable baseline、`FR-0403` collection public behavior 与 `v1.2.0` resource governance foundation 不变。

## 明确不在范围

- 不关闭 Phase `#381`。
- 不交付 `#405` creator profile / media asset read contract。
- 不交付 batch execution、dataset sink、scheduled execution、provider selector、fallback、marketplace 或上层内容库工作流。
- 不提交 raw fixture payload files，不记录外部项目名或本地路径。

## 目标判据

- FR `#404` formal spec suite 已冻结 comment target、page continuation、reply cursor、comment item envelope、visibility status、root/parent/target linkage、dedup key、source trace、raw payload ref 与公共错误分类。
- runtime 已能用统一 comment envelope 承载 `comment_collection`。
- TaskRecord durable truth、result query 与 compatibility consumers 已迁移到 comment public surface。
- sanitized evidence artifact 可复验证明 top-level page、next page、reply page、empty、deleted/invisible/unavailable、duplicate、cursor invalid、permission/rate/platform/provider blocked、partial/credential/verification 边界成立。
- `content_detail_by_url` baseline 与 `FR-0403` collection public behavior 回归保持通过。
