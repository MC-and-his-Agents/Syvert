# Release v0.4.0

## 目标

- 建立最小资源系统，让 Core 而不是 Adapter 承担资源生命周期、资源包注入与任务级资源追踪的运行时语义。

## 明确不在范围

- 不提供完整资源控制台或资源查询 UI/API
- 不引入高级健康恢复循环
- 不纳入浏览器资源管理
- 不展开跨租户策略复杂性或资源能力抽象扩展

## 目标判据

- `FR-0010` formal spec 与 runtime 已冻结并落地账号/代理资源模型、`acquire / release` 接口与 `AVAILABLE / IN_USE / INVALID` 最小状态集合
- `FR-0011` formal spec 与 runtime 已冻结并落地 task-bound tracing truth、资源使用日志与最小时间线投影
- `FR-0012` formal spec 与 runtime 已冻结并落地 Core 注入资源包、Adapter 禁止自行来源化执行资源的边界
- 真实参考适配器仍沿 Core 统一资源路径执行，不绕过 lifecycle 或 tracing truth
- 资源使用可以按任务回放与追踪
