# Release v1.5.0

## 目标

- 发布 `#405` creator profile / media asset read slice，完成 `creator_profile_by_id` 与 `media_asset_fetch_by_ref` 的 formal spec、runtime carriers、consumer migration、sanitized evidence 与 closeout truth。
- 保持 `content_detail_by_url` stable baseline 与 `v1.3.0` collection envelope 不变；`#404` comment collection 由独立 release slice 负责。
- 明确 Phase `#381` 不因本 release planning 自动关闭；Phase closeout 只能由独立 closeout decision 判断。

## 明确不在范围

- 不交付 batch execution、dataset sink、scheduled execution、provider selector、fallback、marketplace 或上层内容库工作流。
- 不交付 media upload、content publish、asset library、media storage lifecycle、thumbnail/transcode pipeline 或 creator profile product model。
- 不提交 raw fixture payload files，不记录外部项目名或本地路径。
- 不把整个 Phase 3 绑定到 `v1.5.0`。

## 目标判据

- FR `#405` formal spec suite 已冻结 creator target、public profile result、media asset ref、media fetch result、content type、fetch policy、fetch outcome、source trace、raw payload ref、normalized result 与公共错误分类。
- canonical taxonomy / Adapter requirement / Provider offer / compatibility decision admission 输入域已显式扩展到 `creator_profile_by_id` 与 `media_asset_fetch_by_ref` 的 approved execution slices。
- runtime 已能承载 `creator_profile_by_id` 与 `media_asset_fetch_by_ref` 的公共 result carriers。
- TaskRecord durable truth 与 result query consumers 已迁移到 creator/media public surface；compatibility decision 只匹配 approved requirement / offer inputs，不消费 result carriers。
- sanitized evidence artifact 可复验证明 public profile success、creator not found/unavailable、image/video media ref、metadata-only、source-ref-preserved、download/no-download、media unavailable、unsupported content type、permission/rate/platform/provider blocked、parse_failed、signature_or_request_invalid、fetch_policy_denied、credential/verification 边界成立。
- release closeout 必须复验 `content_detail_by_url` 与 `#403` collection envelope 回归；`#404` shared runtime/consumer conflict-risk clearance 必须在 `#422/#423` runtime Work Item 前提供可追溯 locator，或由 `#426` closeout 明确记录未满足状态。
