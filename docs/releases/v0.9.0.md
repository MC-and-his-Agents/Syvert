# Release v0.9.0

## 目标

- 用 external provider sample evidence 压测 `v0.8.0` 冻结的 Adapter / Provider compatibility decision。
- 证明 `AdapterCapabilityRequirement -> ProviderCapabilityOffer -> AdapterProviderCompatibilityDecision -> Adapter-bound execution evidence` 链路可执行、可审计、可 fail-closed。
- 生成可被 `FR-0351` 的 `provider_compatibility_sample` gate item 消费的仓内 evidence。

## 明确不在范围

- 不正式支持任何指定 provider 产品、SLA 或 marketplace listing。
- 不引入 Core provider registry、provider selector、fallback、priority 或 ranking。
- 不扩展 search / list / comment / batch / dataset 或发布能力。
- 不把上层应用作为 Syvert 主仓交付物。
- 不把 Python package publish 作为本 release 必需交付物。

## 目标判据

- `FR-0355` formal spec 已合入 main。
- external provider sample evidence 已合入 main，并明确 `provider_support_claim=false`。
- matched / unmatched / invalid_contract decision matrix 可复验。
- Adapter-bound execution evidence 覆盖 success raw payload、normalized result、provider error mapping、resource lifecycle release reason 与 observability carrier。
- provider identity 不进入 Core routing、registry discovery、TaskRecord、resource lifecycle、resource trace 或 Core-facing failed envelope。
- 双参考 `content_detail_by_url`、第三方 Adapter-only entry、API / CLI same Core path 持续通过。
