# Release v0.7.0

## 目标

- 稳定 Syvert 适配器表面，使第三方 adapter 作者可以面向清晰的 Adapter contract、SDK 版本标识、兼容性声明与能力元数据开发。
- 在仓内拆清 `Syvert Adapter -> Syvert-owned Provider Port -> Native Provider` 边界，把当前小红书、抖音 adapter 内部的 HTTP/sign/browser bridge 收敛为 native provider 实现细节。
- 保持 Core 只消费 Adapter 运行时契约，不感知 provider 实现、外部工具或 provider 选择策略。

## 明确不在范围

- 不接入 WebEnvoy、OpenCLI、bb-browser、agent-browser 或其他外部 provider。
- 不新增小红书/抖音搜索、评论、账号、发布、通知、互动等业务能力。
- 不引入 Core 级 provider registry、provider selector 或跨 provider fallback 策略。
- 不拆出 adapter 独立仓库。

## 目标判据

- `FR-0021` 冻结 adapter-owned provider port 边界，并明确 native provider 拆分不改变 Core / Adapter contract。
- 小红书、抖音当前已批准 `content_detail_by_url` 验证切片保持兼容，不新增业务操作。
- adapter capability metadata、SDK 版本标识、兼容性声明与迁移约束能共同解释 provider port 对 adapter 作者的影响。
- GitHub Phase / FR / Work Item、release / sprint 索引、PR 与主干事实最终保持一致。
