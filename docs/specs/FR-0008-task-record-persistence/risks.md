# FR-0008 风险清单

## 风险项

| 风险 | 影响 | 缓解策略 | 回滚策略 |
| --- | --- | --- | --- |
| 终态状态已写入，但终态结果或必需日志未可靠落盘 | 查询面会看到不可信的“成功任务”，破坏状态/结果/日志一致性 | formal spec 明确要求终态写入失败时 fail-closed，禁止把该任务声明为 durable success | 通过独立 revert PR 回退任务记录持久化实现，恢复为未启用持久化闭环的上一版本行为 |
| CLI 或外部调用方旁路 Core 直接写入结果文件 | 会出现影子 schema、影子状态与主路径漂移，后续查询无法统一 | formal spec 明确规定持久化必须通过 Core 内部机制；实现侧只暴露统一任务记录写入口 | 通过独立 revert PR 撤销旁路写入入口，并删除影子落盘逻辑 |
| 共享序列化与回读逻辑不一致 | 会出现“写得出、读不回”或不同调用方读取出不同语义 | formal spec 要求以同一份共享 contract 做序列化/反序列化，并对非法记录 fail-closed | 通过独立 revert PR 回退新序列化形状，恢复到上一版共享 contract |
| 平台私有字段渗入共享任务记录 | Core 任务记录会被单一平台绑死，破坏插件式适配器边界 | formal spec 只允许请求快照复用共享输入轴，终态结果继续复用共享 envelope；平台私有语义不得扩张为持久化字段 | 通过独立 revert PR 删除平台专属字段，并重建受污染的持久化样例/测试 |

## 合并前核对

- [ ] 高风险项已有缓解策略
- [ ] 回滚路径可执行
