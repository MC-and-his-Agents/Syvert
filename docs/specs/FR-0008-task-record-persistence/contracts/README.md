# task-record contract（v0.3.0）

## 接口名称与版本

- 接口名称：`task-record`
- contract 版本：`v0.3.0`
- 作用：定义 `FR-0008` 共享任务记录聚合、共享序列化与回读边界

## 输入输出结构

- 输入结构：
  - 输入对象是同一条 `TaskRecord` 聚合，至少包含 `task_id`、请求快照、任务状态、时间戳、执行日志，以及终态时的结果 envelope
  - 请求快照继续复用共享请求语义，不为持久化额外引入平台专属字段
- 输出结构：
  - 输出是 JSON-safe 的稳定表示，供本地稳定存储写入与回读
  - 终态结果继续复用既有 Core envelope：成功态是 `status=success` 的共享结果，失败态是 `status=failed` 的共享错误结果
  - 非终态记录只表达共享任务状态与日志，不伪造终态 envelope

## 错误与边界行为

- 缺失必需字段、字段类型不合法、终态与结果不一致、日志顺序不可信或结果不可序列化时，必须判定为非法任务记录并 fail-closed
- 终态状态、终态结果与必需日志必须被视为同一条 durable contract；其中任一必需部分无法可靠写入时，不得对外宣称任务已经以持久化完成态收口
- 查询消费者只能消费该共享 contract；不允许 CLI 或外部调用方维护影子状态表、影子结果文件或另一套私有 payload

## 向后兼容约束

- `FR-0008` 复用 `FR-0002`、`FR-0004`、`FR-0005` 已批准的共享 envelope / 输入 / 错误语义，不在持久化层重写这些 contract
- `v0.3.0` 不冻结唯一文件布局、唯一目录命名或唯一嵌入式存储引擎；实现可替换，但共享任务记录 contract 不可漂移
