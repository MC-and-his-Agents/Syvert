# FR-0007 风险清单

## 风险项

| 风险 | 影响 | 缓解策略 | 回滚策略 |
| --- | --- | --- | --- |
| 把版本 gate 误写成某个脚本或某条 CI workflow 的接口文档 | formal spec 失去实现弹性，后续实现被无谓锁死 | 只冻结检查对象、触发语义、失败语义与依赖关系，不冻结具体实现入口 | 回退 spec 中对脚本、命令或目录结构的硬编码描述，恢复 requirement-only 口径 |
| 把 `FR-0007` 与 `FR-0006` 混写 | harness 基座与版本级 gate 职责漂移，后续 Work Item 边界失控 | 在 spec 中显式声明 `FR-0006` 负责 harness，`FR-0007` 负责版本 gate 消费与额外回归/泄漏检查 | 回退超出 `FR-0007` 边界的 harness 语义，交回 `FR-0006` formal spec 处理 |
| 平台泄漏边界定义过窄或过宽 | 过窄导致 Core 被平台语义污染，过宽导致 adapter 私有实现被错误拦截 | 只把 Core 主路径与共享模型/共享契约纳入禁止区，保留 adapter 私有实现与平台研究文档的合法平台语义空间 | 回退错误的禁止边界定义，并重新审查允许区 / 禁止区划分 |
| 上游共享模型与 registry 依赖未写清 | 后续实现无法判断 gate 覆盖对象与失败归属 | 在 spec 中明确依赖 `FR-0004` 输入模型、`FR-0005` 错误模型/registry、`FR-0006` harness 结果 | 回退含糊依赖表述，补齐依赖字段与验证对象描述后重新审查 |
