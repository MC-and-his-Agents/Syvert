# Syvert Vision

## 愿景

Syvert 的目标，不是成为又一个堆满平台实现的爬虫项目。

它要成为一个稳定的采集底座：用统一的任务、资源和结果契约，承载不断演进的适配器生态，并让平台接入从一次性工程，逐步变成可重复、可验证、可扩展的标准化工作。


## 为什么存在

今天的大多数采集系统，仍然围绕具体平台堆叠：

- 每接一个新平台，就重写一套执行流程，
- 每加一种采集方式，就复制一套资源管理逻辑，
- 每次平台变化，都让任务、结果和治理边界一起被拖着改，
- 真正可复用的部分很少，真正稳定的部分更少。

这会导致两个结果：

- 平台能力增长越快，系统越容易失控；
- 想扩展社区和自动化能力时，没有稳定宿主可以依附。

Syvert 存在，就是为了解决这个结构性问题。


## Syvert 是什么

Syvert 是一个任务驱动、适配器可插拔的采集底座。

它的核心职责不是描述某个平台怎么抓，而是定义一套稳定的运行时，使不同平台的采集实现可以在同一套规则下被接入、执行、治理、审计和演进。

这意味着：

- Core 负责运行时语义，
- Adapter 负责平台语义，
- 平台差异被收敛到边界之外，而不是持续侵入核心。

Syvert 首先会以媒体和内容平台作为验证场景，但它的底层能力不应被限定在某一类平台之内。


## Syvert 不是什么

Syvert 不应被理解为：

- 某几个平台实现的打包集合，
- 一个只有抽象接口、没有运行时能力的 SDK，
- 一个把不同爬虫脚本集中摆放的仓库，
- 一个以内置平台数量作为核心卖点的产品，
- 一个承诺“自动接入任意网站”的夸大化工具。

它的价值，不在于默认自带多少平台，而在于能否稳定承载更多平台接入。


## 核心原则

- 平台逻辑属于适配器，不属于 Core。
- Core 的成长方向是运行时能力，而不是平台覆盖数量。
- 采集实现技术可以多样，但系统契约必须统一。
- 原始结果与标准化结果都应被保留，前者用于审计与回放，后者用于消费与治理。
- 少量真实适配器比大量抽象讨论更能验证 Core。
- 人与 agent 都应能围绕稳定契约高效构建和维护适配器。


## 面向谁

Syvert 服务三类核心使用者：

- 需要稳定接入多个平台的系统开发者，
- 专注于具体平台接入的适配器开发者，
- 希望借助工具链或 agent 加速适配器生产与验证的团队。

对第一类人，Syvert 提供稳定宿主，避免为每个平台重复建设任务系统、资源系统和结果链路。

对第二类人，Syvert 提供明确边界，使工作重心回到平台接入本身，而不是重复实现一整套运行时。

对第三类人，Syvert 提供逐步标准化的探索、验证和接入路径，使适配器开发从一次性工程转向可复用流程。


## 核心价值

Syvert 希望长期提供的价值，不是“替你抓某个平台”，而是：

- 让不同平台接入遵循同一套运行契约，
- 让采集任务具备统一的执行和治理方式，
- 让账号、代理、浏览器等运行资源进入可管理边界，
- 让结果输出具备统一的审计、回放和标准化基础，
- 让新平台接入的成本随着生态演进持续下降。

如果这些价值成立，适配器就会从系统本体，退回到插件角色。


## 生态方向

Syvert 的长期形态，不应只是一个核心仓库。

它应逐步形成一个围绕 Core 展开的生态：

- Core 提供稳定运行时，
- Adapter SDK 提供稳定接入契约，
- 参考适配器用于验证边界，
- 外部适配器用于扩展平台能力，
- 探索与验证工具链用于降低适配器生产成本。

在这个生态里，平台接入不再只能依赖少数维护者手工推进。

它应逐步变成一种可由开发者协作、也可由 agent 协助完成的标准化工作。


## 长期目标

Syvert 的长期目标，是让平台接入从“每次都重做一遍的工程项目”，演进为“围绕稳定 Core 和稳定契约持续生成、验证和维护的能力体系”。

这不意味着幻想式的全自动化。

更现实也更有价值的目标是：

- 人类开发者可以更快地构建适配器，
- agent 可以更快地产出候选适配器和验证结果，
- 系统可以更稳地承载这些适配器长期运行。

当这三件事同时成立时，Syvert 才真正具备生态价值。


## 成功标准

以下现象出现时，说明 Syvert 的愿景正在成立：

- 新平台主要通过新增适配器完成接入，而不是改写 Core 主链路；
- 不同采集方式仍能共享同一套任务、资源和结果边界；
- 参考适配器能够持续暴露并验证 Core 的抽象质量；
- 使用者评价它的核心价值是“稳定宿主”和“接入效率”，而不是“内置了几个平台”；
- 适配器的生产与维护开始出现明显的标准化和工具化趋势；
- agent 参与适配器开发时，产物是可验证、可维护的接入实现，而不是一次性代码片段。


## 一句话定位

Syvert 是一个任务驱动、适配器可插拔的采集底座，用稳定的 Core 承载平台接入，并让适配器的构建、验证和演进逐步标准化。
